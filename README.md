# Кластеризация сигналов сцинтилляционного детектора

Проект посвящён кластеризации сигналов сцинтилляционного детектора методами машинного обучения без учителя. Цель — разделить 23 479 сигнала на три группы: два основных физических типа частиц и отдельный класс аномальных или неоднозначных сигналов.

## Задача

Необходимо построить воспроизводимое решение для соревнования Kaggle Signal Types Classification. Итоговый файл submission.csv должен содержать метки трёх кластеров:

- кластер 0 — первый тип частиц;
- кластер 1 — второй тип частиц;
- кластер 2 — аномальные, смешанные или неидентифицируемые сигналы.

Решение оформлено в ноутбуке с EDA, предобработкой данных, feature engineering, сравнением моделей, подбором гиперпараметров, выбором финальной модели и интерпретацией результатов.

## Данные

В работе используется файл Run200_Wave_0_1.txt. Каждая строка соответствует одному сигналу, а временная форма сигнала задаётся набором дискретных отсчётов. После удаления служебных столбцов используется 500 временных каналов на сигнал.

Файл с данными нужно положить в одну папку с ноутбуком. Ноутбук также проверяет альтернативные пути к файлу, чтобы решение было удобнее запускать локально и в Kaggle-среде.

## Краткое описание решения

### 1. EDA

В ноутбуке проверяются размерность датасета, пропуски, диапазоны значений, распределения амплитуд, интегралов и положения пика. По данным видно, что основной максимум сигнала расположен около 150-го временного канала, а распределения амплитуд и интегралов имеют выраженные длинные хвосты.

### 2. Предобработка

Сигналы приводятся к положительной шкале с учётом baseline offset. Для дальнейшего обучения используются не исходные 500 каналов напрямую, а интерпретируемые признаки формы сигнала. Масштабирование выполняется через RobustScaler, потому что он устойчивее к выбросам и длинным хвостам распределений.

### 3. Feature Engineering

Для каждого сигнала извлекаются признаки, связанные с физической формой импульса:

- амплитуда пика;
- суммарный интеграл;
- интеграл в окне около пика;
- PSD-доля хвостовой части импульса;
- оценка времени экспоненциального затухания;
- длительность сигнала;
- временной центр масс;
- ширина импульса;
- отношение сигнал/шум;
- асимметрия и эксцесс формы.

Основной акцент сделан на PSD-признаках и параметрах хвоста импульса, так как они помогают разделять разные типы частиц при похожих амплитудах.

### 4. Сравнение моделей

В решении сравниваются несколько подходов:

- IsolationForest + KMeans;
- KMeans на 3 кластера;
- GaussianMixture на 3 компоненты;
- BIRCH на 3 кластера.

Для моделей перебираются гиперпараметры. Качество сравнивается по внутренним метрикам кластеризации: silhouette_score, Calinski-Harabasz и Davies-Bouldin. Также учитывается физическая интерпретируемость кластеров.

### 5. Финальная модель

Финальным выбран подход IsolationForest + KMeans. Сначала IsolationForest выделяет нетипичные сигналы, которые относятся к кластеру 2. Затем KMeans делит оставшиеся нормальные сигналы на два основных физических класса.

Такой подход лучше соответствует постановке задачи, потому что третий класс должен быть не просто третьей плотной группой, а именно группой аномальных, смешанных или неоднозначных сигналов.

В текущем запуске итоговое распределение кластеров получилось следующим:

| Кластер | Количество сигналов | Интерпретация |
|---:|---:|---|
| 0 | 9544 | Один из двух основных типов частиц |
| 1 | 12995 | Второй основной тип частиц |
| 2 | 940 | Аномальные или неоднозначные сигналы |

## Как запустить проект

1. Склонировать репозиторий или скачать файлы проекта.
2. Положить файл Run200_Wave_0_1.txt в папку с ноутбуком.
3. Установить зависимости:

```bash
pip install numpy pandas matplotlib scipy scikit-learn notebook
```

4. Открыть ноутбук:

```bash
jupyter notebook signal_clustering_edited_requirements_final_split.ipynb
```

5. Последовательно выполнить все ячейки.
6. После запуска будет создан файл submission.csv для отправки на Kaggle.

## Воспроизводимость

Для воспроизводимости зафиксирован RANDOM_STATE = 42. Внутренние метрики считаются на фиксированной подвыборке объектов, поэтому результаты сравнения моделей не зависят от случайного состояния предыдущих ячеек.

## Итоговый вывод

В проекте выполнены EDA, предобработка, генерация физических признаков сигнала, сравнение нескольких моделей и подбор гиперпараметров. Итоговый метод IsolationForest + KMeans выбран как наиболее согласованный с постановкой задачи: два кластера описывают основные типы частиц, а третий кластер выделяет аномальные и неоднозначные сигналы.
